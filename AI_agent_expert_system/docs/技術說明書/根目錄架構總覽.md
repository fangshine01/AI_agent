## 🔄 核心機制

### BYOK 認證流程 (Bring Your Own Key)

為符合企業數據安全與限額規範，v2.2.0 以後的系統將「取消內建預設 API Key」。
1. 使用者開啟網頁後，須於在 Streamlit 側邊欄填入個人或單位核發的 API Key。
2. 進行驗證時，此 Key 將先戳至 **Corporate API Proxy** 驗證合法性並解析 User Hash。
3. 驗證通過後，Key 會保存在記憶體中（Session State），後續每一個 RAG、Chat、或是上傳向量化操作，均以標頭 (Header) 夾帶該 Key 傳遞至 FastAPI 後端。

### 跨平台代理整合 (27 款模型)

系統於 `backend/config.py` 配置之可用模型多達 27 款，包含：
- **OpenAI 陣營**: `gpt-4o`, `o1-mini`, `gpt-4.1` 等
- **Google 陣營**: `gemini-2.5-flash`, `gemini-3-pro-preview` 等
- **Azure 陣營**: `azure-gpt-4o`, `azure-o1-mini` 等

使用者無論在介面上下拉選擇何種模型引擎，核心向後發送的單一端點皆為：`http://innoai.cminl.oa/agency/proxy/openai/platform` (公司反向代理層)。透過傳入不同的 `model_id`，代理層便會將問題配發到對應的大型語言模型供應商進行推論。

### 對話 Session 機制

- 每當創建一次對話，後端建立一筆帶有 UUID 的 `session_id`。
- 回答過程所消耗的 Token 、輸入內容與產出，皆於生成後透過背景線程 (Background Tasks) 隱含寫入 `session_db.py` 中。
- 左側側邊面板可動態取得 `GET /session` 清單幫助使用者隨時繼續先前的探討流程。

## 🛠️ 開發與編譯起手式

1. **安裝依賴（前後端切割）**:
   ```bash
   pip install -r backend/requirements.txt
   pip install -r frontend/requirements.txt
   ```
2. **複製環境範本**: `cp .env.example .env` (後端)
3. **一鍵執行批次檔 (Windows)**:
   ```cmd
   Start_all.bat
   ```
   腳本背景包含 `uvicorn` (port 8000) 以及 `streamlit run` (port 8501)。
