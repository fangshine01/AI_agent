## 3. 資料流

### 3.1 查詢流程 (BYOK 認證版)

```text
使用者在 Streamlit 驗證 API Key 並提交問題
    ↓
Streamlit 取得 `session_id`，發出附加驗證標頭的 HTTP POST 到 FastAPI
    ↓ (httpx POST /api/v1/chat/query)
FastAPI Chat Router 驗證 Token 合法性
    ↓
dependencies.py → get_search()
    ↓
core/search/ 執行向量搜尋 (sqlite-vec) + 關鍵字搜尋
    ↓
dependencies.py → get_ai_core()
    ↓
core/ai_core.py 組合 Prompt 以及選擇的模型 (`gpt-4o`, `gemini-2.5-flash` 等)
    ↓
透通企業 API Proxy 向 OpenAI/Google/Azure 請求
    ↓
生成結果寫入 Session 對話記錄 DB，並即時返回 JSON 給前端
    ↓
Streamlit 解析並渲染 Markdown / Mermaid 圖表
```

### 3.2 檔案自動入庫與視覺分析流

```text
使用者拖拽上傳 或 手動放入 data/raw_files/
    ↓
Watchdog 監視系統 (Debounce 2s)
    ↓
判定檔案格式 (.md/.txt/.pdf/.pptx 或是 .png/.jpg)
    ↓
┌──────────────┬────────────────────────────┐
│ 文字與投影片  │ 圖片檔案                    │
├──────────────┼────────────────────────────┤
│ core/        │ 呼叫企業 API Proxy (Vision)  │
│ ingestion_v3 │ 轉寫為結構化 Markdown 文本    │
│              │ → 輸出至 generated_md/       │
│              │ → 進入 ingestion 標準流程     │
└──────┬───────┴──────────────┬─────────────┘
       ↓                      ↓
     文本分塊 (Chunks) 與 Embedding 向量化 (text-embedding-3)
       ↓
    SQLite 與 VectorDB 入庫，儲存中繼資料 (Metadata)
       ↓
檔案歸檔至 archived_files/ (或失敗移至 failed_files/)
```

## 6. 核心設計決策

### 6.1 前後端分離 (Streamlit + FastAPI)
原本初期版本全在 Streamlit 中耦合，v2 將 UI 與後端邏輯解除綁定，使得未來如果前端需要更換為 React 或是 Vue，後端 API 均能原封不動繼續使用。

### 6.2 BYOK (Bring Your Own Key) 模式與跨平台支援
為分散企業內 API 額度限制並且區分個人對話紀錄，取消系統共用 API Key 設定。每一位使用者登入時填入 API Key 進行綁定。同時藉由企業 API Proxy，只需同一套介接邏輯即可無縫支援並切換 27 款來自不同供應商的模型。

### 6.3 依賴注入保留 Core 模組獨立性
為了避免全面破壞已經穩定運行的 `core/` 爬蟲與爬梳邏輯，API 路由 (Routers) 使用 `app/dependencies.py` 的依賴注入，藉此調度 `core`，保留了未來的延展性與降低重構風險。
