# AI Expert System 架構總覽 (v2.4.0)

> 本文檔提供針對開發與維護者快速入門本知識庫系統之「全觀概念」。詳細各目錄功能定義及端點列表請移步另外的 `根目錄架構總覽.md` 查閱。

## 系統定位

此 AI Agent 系統不僅僅是普通的對話機器人，而是一個**可動態增長知識的 AI 助理引擎**。它能夠:
1. 自動監視並爬梳丟入特定資料夾的各式檔案（PDF, PPT, Word, 圖檔）。
2. 將這些檔案轉化為 AI 可讀理解的語意向量 (Vector Chunks)。
3. 當工程師詢問廠內技術問題時（SOP、Troubleshooting 等），優先找出資料庫對應片段，並送給 ChatGPT / Gemini 理解後回覆。

## 三層級獨立架構

整套服務嚴格分離為三個板塊運作：

### 1. 互動介面層 (Streamlit Frontend)
它是一個僅專營網頁展示的客戶端程式（預設在 `8501` 埠口運行）。這使得:
- 用戶可以點擊按鈕、下拉選單選擇不同的 AI 模型（27種以上）。
- 處理使用者的身份登入與 API Key 設定 (BYOK 機制)。
- 它**不**直接連線知識庫或是操作檔案系統本身，所有行為都會包裝為純 HTTP 請求發送出去。

### 2. 邏輯閘道層 (FastAPI Backend)
負責接收所有從前端送來的指令，預設於 `8000` 埠口運行，職責包含:
- **認證把關**: 檢查請求中的 API Key 權限。
- **排程分流**: 確認請求是「我要查找答案」還是「我要上傳解析一份新手冊」。
- **監控警報**: 在後台啟用一隻背景程式 (Watcher)，若你沒有透過網頁上傳，而是直接把檔案貼入資料夾內，Watcher 這隻狗就會發現並自動告訴後台進行解析。

### 3. 大腦與儲存層 (Core & SQLite-vec)
被呼叫的實體引擎（位於根目錄 `core/` 下的元件）負責苦力活:
- 使用各種套件幫投影片抓出純文字，如果有圖檔，將圖檔丟入企業內建 API Proxy 請 AI 解讀為文字 (Vision 轉譯)。
- 將一萬字的內容自動依照語意切割成數十個片段。
- 運算向量數值相似度，並將結果存入資料庫內 (`knowledge_v2.db`) 以供下次提問時快速命中相似段落。

## 機制演進亮點

- **告別過時的 v1.50 單體叢集**: 捨棄舊有 `chat_app.py` 及 `admin_app.py` 將前後端混雜寫於同一檔案內的方式。
- **API 代理統一性**: 模型端點已實行反向代理中繼機制，後端呼叫只需定向單一 `innoai.cminl.oa` URL。
- **使用者記錄持久化**: 支援記錄每個人的 Session 對話內容與 Token 消費圖表回朔。
